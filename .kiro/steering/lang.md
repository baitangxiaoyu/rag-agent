# 语言规范

所有 AI 交互、代码产出和文档编写遵循以下规则。

## 对话语言

- 始终使用中文回复用户。
- 技术术语首次出现时使用「中文（English）」格式，后续可直接使用英文缩写。
- 语气简洁专业，避免冗余解释和客套寒暄。

## 代码规则

### 注释与文档

- 代码注释必须使用中文，禁止英文注释。
- 复杂业务逻辑、非显而易见的设计决策处必须添加中文注释说明意图。
- docstring 使用中文描述，参数类型注解使用 Python 类型标注。
- 错误信息和日志消息使用中文。

### 命名与风格

- 标识符（变量名、函数名、类名）使用英文，遵循 PEP 8：
  - 函数和变量：`snake_case`
  - 类名：`PascalCase`
  - 常量：`UPPER_SNAKE_CASE`
- 行长度上限 120 字符（与 Ruff 配置一致）。
- import 排序由 Ruff `isort` 规则自动管理。

### 提交与分支

- Git commit message 使用中文，格式：`<类型>: <简述>`，如 `feat: 新增文档检索接口`。
- 类型包括：feat / fix / refactor / docs / test / chore。
